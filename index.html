Jarvus HTML & CSS Style Guide

General

Indentation

Use 4 spaces. Do not use tabs.

<div>
    <p>Lorem ipsum</p>
</div>
p {
    margin-bottom: 1em;
}

Capitalization

Use lowercase, including on hex values.

Do

<a href="http://jarv.us">Jarvus Innovations</a>
color: #f5f5f5;

Don’t

<A HREF="HTTP://JARV.US">Jarvus Innovations</A>
COLOR: #F5F5F5;

Trailing Whitespace

Remove trailing white space.

Do

<p>Jarvus Innovations</p>
margin-bottom: 1em;

Don’t

<p>Jarvus Innovations</p>   
margin-bottom: 1em;  

Resource Linking

Use relative URLs for local resources. Use protocol relative URLs when appropriate.

Do

<img src="/img/image.jpg" alt="Image." />
div {
    background-image: url(//img/image.jpg);
}
<img src="//img/image.jpg" alt="Image." />

Don’t

<img src="http://jarv.us/img/image.jpg" alt="Image." />
div {
    background-image: url(http://jarv.us/img/image.jpg);
}

Encoding

Encode HTML and CSS documents in UTF-8. Specify this encoding in HTML in the head (not necessary in CSS).

<meta charset="utf-8">

HTML

Document Type

Use the HTML5 doctype:

<!DOCTYPE html>

Language

Specify the document language when appropriate.

<html lang="en-us">

Document Formatting

Line break after every block element. Indent every child element.

<blockquote>
    <p>Lorem ipsum dolor sit amet.</p>
</blockquote>
<table>
    <thead>
        <tr>
            <th>Heading</th>
            <th>Heading</th>
            <th>Heading</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Table cell</td>
            <td>Table cell</td>
            <td>Table cell</td>
        </tr>
    </tbody>
</table>

Optional Tags

Always close tags. Even when optional.

Do

<ul>
    <li>List item</li>
    <li>List item</li>
    <li>List item</li>
</ul>

Don’t

<ul>
    <li>List item
    <li>List item
    <li>List item
</ul>

Quotation Marks

Always use double quotes for attributes.

Do

<p class="statement">Lorem ipsum dolor sit amet.</div>

Don’t

<p class='statement'>Lorem ipsum dolor sit amet.</div>
<p class=statement>Lorem ipsum dolor sit amet.</div>

Linked Resource Types

Leave type attributes off links to stylesheets and scripts

Do

<link rel="stylesheet" href="styles.css">
<script src="script.js"></script>

Don’t

<link rel="stylesheet" href="styles.css" type="text/css">
<script src="script.js type="text/javascript"></script>

Character Entities

Avoid using HTML entities. Use the appropriate unicode characters.

Do

<p>Jeg føler mig dårlig.</p>

Don’t

<p>Jeg f&oslash;ler mig d&aring;rlig</p>

Exception

Use entities for space characters.

<p>I am using a non-breaking space to prevent an&nbsp;orphan.</p>

Semantics

Use elements for their intended purpose.

Do

<p>Lorem ipsum dolor sit amet.</p>
<a href="/about.html">About</div>
<button type="button">Load more</button>

Don’t

<div>Lorem ipsum dolor sit amet.</div>
<div onclick="goToAbout();">About</a>
<a href="#">Load more</a>

Separation of Concerns

Avoid inline styles and javascript.

Don’t

<div style="font-size: 14px;">Lorem ipsum dolor sit amet</div> 
<a href="#" onclick="doSomething();">Lorem</a>

Multimedia Fallbacks

Always include alternative text for multimedia.

Do

<img src="/img/bread.jpg" alt="Rye bread on a table." />

HTML5 Outline

Do not rely on the HTML5 outline algorithm. Use traditional heading structures. Screen readers do not widely recognize the HTML5 outline.

Do

<section>
    <h1>Section Title</h1>
    <p>Lorem ipsum dolor sit amet.</p>
    <p>Lorem ipsum dolor <a href="/home.html">lorem ipsum</a></p>
    <article>
        <h2>Article Title</h2>
        <p>Lorem ipsum dolor sit amet.</p>
        <h3>Article Subtitle</h3>
        <p>Lorem ipsum dolor sit amet.</p>
    </article>
</section>

Don’t

<section>
    <h1>Section Title</h1>
    <p>Lorem ipsum dolor sit amet.</p>
    <p>Lorem ipsum dolor <a href="/home.html">lorem ipsum</a></p>
    <article>
        <h1>Article Title</h1>
        <p>Lorem ipsum dolor sit amet.</p>
        <h2>Article Subtitle</h2>
        <p>Lorem ipsum dolor sit amet.</p>
    </article>
</section>

CSS

Selector Formatting

Selector on the same line as the opening bracket. Style declarations each on their own line. Closing bracket on its own line after the last property.

.selector {
    margin-bottom: 1em;
}

Put one empty line between selectors.

.selector-one {
    margin-bottom: 1em;
}

.selector-two {
    border-color: red;
}

Selector Grouping

Put multiple selectors on their own line.

.selector-one,
.selector-two,
.selector-three {
    margin-bottom: 1em;
}

Property Order

Put all properties in alphabetical order. This prevents creating duplicate properties and simplifies working with multiple developers.

.selector {
    background-color: white;
    color: black;
    display: inline-block;
    margin-bottom: 1em;
    position: relative;
    z-index: 1;
}

Indentation

Indent selectors inside media queries.

@media all and (min-width: 35em) {

    .selector {
        margin-bottom: 1em;
    }

}

Optional (follow the existing code): Indent related selectors based on their parent-child relationship.

.parent {
    margin-bottom: 1em;
}

    .child-one {
        color: black;
    }

        .child-one-child {
            background-color: red;
        }

    .child-two {
        color: white;
    }

Note: The goal is to show related elements of discrete structures and to avoid adding extra specificity only for CSS readability, not to recreate the entire DOM.

Quotation Marks

Use single quotes except on URIs.

Do

.selector {
    background-image: url(/img/graphic.png);
    font-family: 'Times New Roman', serif;
}

Decimals

Round decimals up to five decimal places. Include the preceeding 0 for numbers less than 1.

Do

.selector {
    font-size: 1.66667em;
    margin-left: 0.75em;
}

Units

Do not include units on zero values.

.selector {
    margin-left: 0;
}

Color Values

Use shorthand for hex values when possible.

.selector {
    color: #333;
}

Use named colors for very common color values.

.selector {
    background-color: white;
}

Use RGBA and other CSS3 color values when necessary, but generally avoid for deeper support.

.selector {
    background-color: rgba(0,0,0,0.5);
}
